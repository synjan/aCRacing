Kjente problemer

🚨
stracker-trackday er nede. Mangler UDP_PLUGIN_ADDRESS i /opt/ac-trackday/cfg/server_cfg.ini. Stracker krever denne for a motta hendelser fra AC-serveren. Fix: legg til UDP_PLUGIN_ADDRESS=127.0.0.1:11000 under [SERVER] og restart: systemctl restart stracker-trackday. Strackerloggen viser ogsa ugyldig listening_port.

# Deploy konfig (srv = trackday | mx5cup | gt3)

# Restart / Logger

Viktig a vite
🤖 AI = kun Practice (MODE=1). Race-modus gir "no AI slots available".
📁 AI splines: Leter alltid i tracks/{bane}/ai/, ignorerer CONFIG_TRACK.
📨 WELCOME_MESSAGE = filsti, ikke tekst. Sett tom.
🔒 data.acd (checksum): Hver bil trenger content/cars/{bil}/data.acd pa serveren. Kopier fra lokal AC-installasjon: scp "C:/.../assettocorsa/content/cars/{bil}/data.acd" root@SERVER:/opt/ac-{srv}/content/cars/{bil}/
🌍 Tidssoner for mod-baner: Nye baner trenger gyldig IANA-tidssone i cfg/data_track_params.ini. Uten dette crasher serveren med "Invalid time zone".
🔌 UDP_PLUGIN_ADDRESS ma settes i server_cfg.ini for at stracker skal motta hendelser fra AC-serveren.
Stracker
📊 Kildekode: github.com/gregsqueeb/S-P-Tracker. Klonet til /opt/stracker/src/, kjorer i Python 3.12 venv.

Python 3.12-patcher (allerede utfort):

  • externals/pygal/_compat.py: collections.Iterable → collections.abc.Iterable
  • ptracker_lib/async_worker.py + database.py: async → is_async (reservert nkw i Python 3.7+)
  • stracker.py: PostgresqlBackend import wrappet i try/except (psycopg2 ikke installert)
  • setuptools < 71 pakreves for pkg_resources
